# 調査ページ 設計ドキュメント

**日付:** 2026-04-18  
**対象:** pog-tool (Streamlit アプリ)  
**目的:** 指定条件の馬を複数POG情報サイトから自動調査し、指名リストを絞り込む

---

## 概要

新ページ `pages/5_調査.py` を追加する。  
対象馬を一括スクレイピングし、近況コメント・POGスコア・兄弟実績を一覧＋詳細表示する。  
主目的はアチーブメント（称号）狙い候補の絞り込み。

---

## 調査対象

### タブ1: レイデオロ産駒
- 種牡馬「レイデオロ」の産駒全頭
- `pog_list.xlsx` の `父名 == "レイデオロ"` で抽出

### タブ2: 過去指名牝馬の仔
- `history.json` から `sex == "牝"` の指名馬を全年度分抽出
- `pog_list.xlsx` の `母名` カラムと照合して今年の産駒を特定
- 表示時に母の名前・指名年・指名時ポイントも併記

---

## データ取得設計

### 新モジュール: `research_scraper.py`
既存の `scraper.py` とは完全分離。

#### 取得先と取得内容

| サイト | 取得内容 |
|--------|---------|
| net競馬 (netkeiba.com) | 近況コメント、調教タイム、兄弟賞金実績 |
| うまなりくん | POG注目ランキング・スコア |
| POGの王様 | POG人気順位・コメント |

#### キャッシュ
- `cache/research_reideouro.json` — レイデオロ産駒データ
- `cache/research_past_mares.json` — 過去指名牝馬の仔データ
- キャッシュ有効期限: 24時間（近況情報は毎日更新されるため）
- キャッシュ構造（各馬）:
```json
{
  "horse_name": "馬名",
  "sex": "牡",
  "dam": "母名",
  "trainer": "調教師",
  "stable": "栗東",
  "reg_no": "血統登録番号",
  "achievement_flag": true,
  "netkeiba": {
    "comment": "近況コメント全文",
    "training": "調教タイム",
    "siblings_prize": 12345678
  },
  "umanariku": {
    "rank": 42,
    "score": 88,
    "comment": "注目コメント"
  },
  "pog_ou": {
    "rank": 55,
    "comment": "人気コメント"
  }
}
```

## UI設計

### ページ構成
```
5_調査.py
├── サイドバー: フィルタ（性別 / 称号候補のみ / POG人気順ソート）
├── [データ更新] ボタン（再スクレイピング）
├── タブ1: レイデオロ産駒
│   ├── 一覧テーブル
│   └── expander（詳細）
└── タブ2: 過去指名牝馬の仔
    ├── 一覧テーブル
    └── expander（詳細）
```

### 一覧テーブルのカラム
| カラム | 内容 |
|--------|------|
| 称号 | 🏆（称号候補マーク） |
| 馬名 | |
| 性別 | 牡/牝 |
| 母名 | タブ2では母の指名年も表示 |
| 産地・調教師 | |
| POG人気 | うまなりくん順位 |
| 近況要約 | コメント冒頭50文字 |

### expander 展開時の詳細表示
```
▼ 馬名（性別）
  [net競馬] 近況コメント全文 / 調教タイム / 兄弟賞金実績
  [うまなりくん] POGスコア・コメント
  [POGの王様] 人気順位・コメント
  [➕ マイリストに追加] ボタン（牡/牝リストへ）
```

## エラーハンドリング
- スクレイピング失敗時は該当フィールドを空文字で記録し、表示側で「取得失敗」と表示
- サイト構造変更への対応: 各サイトのパーサーを関数単位で分離し、修正しやすくする
- レート制限対策: サイト間に1秒のsleepを挿入

---

## 制約・注意事項
- レイデオロはPOG期間中のG1馬不在など戦績が難しい種牡馬のため、近況コメント・調教内容を特に重視する
- 過去指名牝馬の仔は称号（アチーブメント）狙いの第一候補として扱う
- スクレイピング対象サイトの利用規約に注意する
